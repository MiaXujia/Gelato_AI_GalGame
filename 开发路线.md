# 阶段一：核心智能体引擎 (MVP - 最小可行性产品)
**目标：在本地终端（Terminal）里跑通全流程，证明核心逻辑可行。**
不需要任何网页或数据库，纯 Python 脚本运行。

## 步骤

- 步骤 1.1：数据结构定义

使用 Pydantic 定义剧本、场景、角色、对话的 JSON 格式（Schema）。

- 步骤 1.2：构建 Writer Agent（编剧）

连接大模型 API（如 OpenAI），让其根据用户输入的自然语言，稳定输出符合上述 Pydantic 格式的 JSON 剧本。

- 步骤 1.3：构建 Artist Agent（美术）

解析编剧生成的 JSON，提取需要绘画的背景和角色。

调用图像生成 API（如 DALL-E 3），并将图片下载到本地。

- 步骤 1.4：构建 Coder Agent（程序）与资源组装

编写 Python 脚本，将基础的 Ren'Py 游戏模板复制一份。

把生成的图片放入对应文件夹，将 JSON 剧本转化为 script.rpy 语法并写入文件。

里程碑： 在命令行输入“一个赛博朋克风格的咖啡馆相遇故事”，几分钟后，本地生成一个可以直接双击运行的 Ren'Py 游戏包。

# 阶段二：后端服务与异步重构 (展示工程能力)
**目标：将本地脚本升级为能处理并发、防超时的 Web 服务。**
“真实世界应用开发”的关键阶段。

## 步骤

- 步骤 2.1：搭建 FastAPI 骨架

初始化 FastAPI 项目，配置路由结构。

- 步骤 2.2：引入 Celery + Redis（异步任务队列）

因为 AI 生成可能需要 3-5 分钟，绝对不能让 API 接口阻塞。把阶段一的“生成任务”封装成 Celery 后台任务。

- 步骤 2.3：数据库与状态管理

配置 SQLite 或 PostgreSQL。

设计数据表：记录任务 ID、当前状态（排队中/生成剧本中/绘画中/已完成）、生成的项目文件路径。

- 步骤 2.4：编写 RESTful APIs

开发 POST /api/generate（提交生成任务）。

开发 GET /api/status/{task_id}（查询任务进度）。

里程碑： 可以用 Postman 调用接口提交任务，并能不断查询到任务的状态更新，最终获得游戏压缩包的下载链接。

# 阶段三：前端交互界面 (提升用户体验)
目标：开发一个漂亮的用户界面，让任何人都能方便地使用你的作品。

- 步骤 3.1：前端项目初始化

使用 Next.js 或 Vue 3 搭建脚手架。

- 步骤 3.2：开发输入控制台

制作一个表单，让用户输入故事大纲、选择游戏风格（二次元/写实/像素）、设定时长等。

- 步骤 3.3：开发实时状态看板（核心亮点）

前端轮询（或使用 WebSocket）请求后端的状态接口。

在界面上生动地展示 Agent 的工作流，例如亮起指示灯：“✍️ 编剧 Agent 正在构思... -> 🎨 美术 Agent 正在绘制背景... -> ⚙️ 正在打包引擎...”。

里程碑： 拥有一个完整的 Web App，可以在浏览器中完成所有操作。

# 阶段四：部署与简历打磨 (冲刺实习 Offer)
目标：将项目上线，并提炼成简历上的高光项目。

- 步骤 4.1：容器化 (Docker)

编写 Dockerfile 和 docker-compose.yml，一键启动前端、后端、Redis 和数据库。

- 步骤 4.2：云端部署

将后端部署到云服务器（如 AWS EC2 或阿里云），前端部署到 Vercel 或 Netlify。

- 步骤 4.3：文档与开源展示

编写高质量的 README.md，包含架构图、Agent 协作流程图、动图演示 (GIF)。
